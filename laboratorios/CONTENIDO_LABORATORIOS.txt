LAB 4: AMAZON EC2 Y SECURITY GROUPS
------------------------------------
Duración: 90-120 minutos | Dificultad: Intermedia | Páginas estimadas: ~25

Objetivos:
- Lanzar instancias EC2 en subredes públicas y privadas
- Configurar Security Groups (firewall stateful)
- Implementar bastion host para acceso seguro
- Probar conectividad real a través de IGW

Contenido a desarrollar:
- Marco teórico: EC2 conceptos, tipos de instancia (t2.micro Free Tier)
- Security Groups: stateful firewall, reglas inbound/outbound
- Key pairs para SSH, mejores prácticas de seguridad
- Procedimiento:
  1. Crear Key Pair para SSH
  2. Lanzar EC2 t2.micro en Public-Subnet-1A (bastion/jump host)
  3. Lanzar EC2 t2.micro en Private-Subnet-1A (servidor backend)
  4. Configurar Security Groups:
     - Bastion-SG: permitir SSH desde tu IP
     - Private-SG: permitir SSH solo desde Bastion-SG
  5. Conectar a bastion, luego saltar a instancia privada
  6. Verificar conectividad internet desde pública, no desde privada
- 15 preguntas sobre EC2, Security Groups, arquitectura bastion

Servicios: EC2 (t2.micro Free Tier 750hrs/mes), Security Groups, Key Pairs
Costo: $0.00 (Free Tier)

LAB 6: VPC PEERING - CONECTIVIDAD ENTRE VPCs
---------------------------------------------
Duración: 75 minutos | Dificultad: Intermedia | Páginas estimadas: ~20

Objetivos:
- Conectar dos VPCs mediante VPC Peering
- Configurar routing entre VPCs
- Comprender limitaciones (no transitivo, no overlapping CIDR)
- Casos de uso: Dev/Prod separation

Contenido a desarrollar:
- Marco teórico: VPC Peering concepts, topologías, limitaciones
- Peering no es transitivo: A-B y B-C no implica A-C
- CIDR ranges no deben solaparse
- Procedimiento:
  1. Crear segunda VPC: VPC-B 172.16.0.0/16 con 2 subnets
  2. Lanzar EC2 en VPC-B
  3. Crear VPC Peering Connection (requestor VPC-A, acceptor VPC-B)
  4. Aceptar peering request
  5. Agregar rutas en ambas VPCs: A → B CIDR via pcx-xxx, B → A CIDR via pcx-xxx
  6. Actualizar Security Groups para permitir tráfico
  7. Probar conectividad ping entre instancias de ambas VPCs
- 15 preguntas sobre peering, routing, casos de uso

Servicios: VPC Peering, Multiple VPCs, Route Tables
Costo: $0.00 (VPC peering gratuito, solo transferencia datos intrarregión)

LAB 8: PROYECTO INTEGRADOR - ARQUITECTURA COMPLETA 3-TIER
----------------------------------------------------------
Duración: 120-150 minutos | Dificultad: Avanzada | Páginas estimadas: ~30

Objetivos:
- Integrar todos los conceptos de Labs 1-7
- Diseñar e implementar arquitectura 3-tier completa
- Aplicar mejores prácticas de seguridad y alta disponibilidad
- Documentar arquitectura end-to-end

Escenario:
"Implementar aplicación web 3-tier en AWS con separación de capas, seguridad multicapa, y monitoreo completo usando Free Tier"

Arquitectura a implementar:
- Capa Presentación: Instancias EC2 web servers en subnets públicas multi-AZ
- Capa Aplicación: Instancias EC2 app servers en subnets privadas multi-AZ
- Capa Datos: Simulación de DB en subnet privada (sin RDS para Free Tier)
- Bastion host para administración segura
- Load balancing conceptual (sin crear ALB para evitar costos)

Procedimiento:
1. Verificar infraestructura Labs 1-3: VPC, subnets, IGW, routes
2. Lanzar 2 EC2 web (Public-Subnet-1A y 1B) con Apache/Nginx
3. Lanzar 2 EC2 app (Private-Subnet-1A y 1B) con app simulada
4. Configurar Security Groups:
   - Web-SG: allow HTTP 80, HTTPS 443 desde 0.0.0.0/0
   - App-SG: allow puerto app (8080) solo desde Web-SG
   - DB-SG: allow puerto DB (3306/5432) solo desde App-SG
   - Bastion-SG: allow SSH desde tu IP
5. Configurar NACLs para cada capa
6. Habilitar VPC Flow Logs completos
7. Crear CloudWatch dashboard con métricas de toda la arquitectura
8. Configurar alarmas para cada capa
9. Documentar: diagrama arquitectura, IPs, Security Groups, rutas
10. Calcular costos reales vs Free Tier

Entregables del proyecto:
- Diagrama de arquitectura completo (3-tier + seguridad + monitoreo)
- Tabla de recursos creados (VPC, subnets, EC2, SGs, NACLs, etc.)
- Configuración de Security Groups y NACLs documentada
- Tabla de routing completa
- Screenshots CloudWatch dashboard (excepción: único lab con capturas)
- Análisis VPC Flow Logs
- Cálculo de costos (debe ser $0.00 Free Tier)
- 20 preguntas integradoras sobre toda la arquitectura
- Conclusiones: lecciones aprendidas, mejores prácticas aplicadas

Servicios integrados:
- Todos los anteriores: VPC, Subnets, IGW, Route Tables
- EC2 (6 instancias t2.micro máximo)
- Security Groups, NACLs
- VPC Flow Logs, CloudWatch, SNS

Costo: $0.00 (Free Tier: 750 hrs EC2/mes = 31 días * 24hrs, suficiente para 6 instancias corriendo 5 días)
