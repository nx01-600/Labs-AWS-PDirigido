LAB 1: INTRODUCCIÓN A AWS Y CONFIGURACIÓN INICIAL
-------------------------------------------------
Duración: 60-90 minutos | Dificultad: Básica

Objetivos:
- Crear cuenta AWS con alertas de facturación
- Configurar IAM (usuarios, grupos, políticas)
- Habilitar MFA para seguridad
- Familiarización con consola AWS

Contenido implementado:
- Marco teórico: Cloud Computing, IaaS/PaaS/SaaS, AWS global infrastructure
- Procedimiento: Registro cuenta, verificación email/teléfono/pago, IAM setup completo
- Usuario IAM + Grupo Administradores + Política AdministratorAccess
- MFA para root y usuario IAM
- CloudWatch billing alerts con SNS
- 15 preguntas evaluativas con respuestas justificadas

Servicios: IAM, Billing Dashboard, CloudWatch, SNS
Costo: $0.00


LAB 2: AMAZON VPC - REDES VIRTUALES
------------------------------------
Duración: 75-90 minutos | Dificultad: Básica-Intermedia

Objetivos:
- Crear VPC personalizada con arquitectura multi-AZ
- Implementar subnets públicas y privadas
- Configurar tablas de enrutamiento
- Comprender CIDR y segmentación de red

Contenido implementado:
- Marco teórico: VPC concepts, CIDR notation, subnet design, routing
- Arquitectura: VPC 10.0.0.0/16 con 4 subnets en 2 AZs
  * Public-Subnet-1A: 10.0.1.0/24 (us-east-1a)
  * Public-Subnet-1B: 10.0.2.0/24 (us-east-1b)
  * Private-Subnet-1A: 10.0.11.0/24 (us-east-1a)
  * Private-Subnet-1B: 10.0.12.0/24 (us-east-1b)
- Tablas de enrutamiento: Public-Route-Table y Private-Route-Table (Main)
- Auto-assign public IP habilitado en subnets públicas
- 15 preguntas evaluativas sobre VPC, subnetting, routing

Servicios: VPC, Subnets, Route Tables, Availability Zones
Costo: $0.00


LAB 3: INTERNET GATEWAY - CONECTIVIDAD A INTERNET
--------------------------------------------------
Duración: 45-60 minutos | Dificultad: Intermedia

Objetivos:
- Implementar conectividad bidireccional a internet
- Configurar Internet Gateway y routing
- Diferenciar subredes públicas vs privadas operacionalmente
- Comprender NAT automático de AWS

Contenido implementado:
- Marco teórico: IGW funcionamiento, NAT translation, IP público/privado (RFC 1918)
- IGW vs NAT Gateway comparison, traffic flows, security best practices
- Procedimiento: Crear IGW (estado detached), adjuntar a VPC (relación 1:1)
- Agregar ruta 0.0.0.0/0 → IGW en Public-Route-Table
- Verificar auto-assign public IP en subnets públicas
- Mantener Private-Route-Table sin ruta por defecto (solo local)
- 15 preguntas sobre conectividad, NAT, arquitecturas público/privado

Servicios: Internet Gateway, Route Tables, Elastic IPs (conceptual)
Costo: $0.00 (IGW gratuito, primeros 100GB transferencia/mes gratis)
